## 1. Yohaku 设计哲学

### 1.1 核心理念

> 「留白也是写作的一部分」—— 画面里那些有意空出的地方，往往比填满的部分更有分量。

Yohaku（余白）取自日语「留白」，最初是为个人博客设计的视觉体系。BetterWrite 将其核心理念扩展为一套完整的应用设计系统：

- **颜色是克制的** — 浅色模式接近真实纸张的米白，深色模式沉入暖灰
- **动画是呼吸式的** — 元素随滚动自然浮现，而非弹出
- **字体是有质感的** — 标题用衬线字体，注释以斜体呈现
- **交互是低调的** — 悬停时仅颜色微微加深，如纸面被指尖轻触

### 1.2 设计原则

```
1. 内容优先 (Content First)
   └── 设计服务于内容，排版和布局优先考虑可读性

2. 留白呼吸 (Breathing Whitespace)
   └── 充足的留白让内容"呼吸"，减少认知负担

3. 安静精致 (Quiet Sophistication)
   └── 避免花哨效果，追求时间感、专业感

4. 纸感体验 (Paper-like Experience)
   └── 模拟纸张的阅读体验，尤其适合写作场景
```

## 5. 响应式断点

```css
/* Tailwind CSS 4 断点 */
/* 基于 Yohaku 设计，内容优先 */

@theme {
  --breakpoint-sm: 640px;   /* 手机横屏 */
  --breakpoint-md: 768px;   /* 平板竖屏 */
  --breakpoint-lg: 1024px;  /* 平板横屏 / 小笔记本 */
  --breakpoint-xl: 1280px;  /* 桌面 */
  --breakpoint-2xl: 1536px; /* 大桌面 */
}
```

### 响应式策略

```
移动端 (< 768px)
├── 单列布局
├── 底部导航栏（Tab Bar）
├── 卡片全宽
├── 作文编辑器全屏
└── 简化版数据图表

平板端 (768px - 1024px)
├── 双列布局
├── 侧边栏可折叠
├── 作文编辑器 + 批改结果 上下布局
└── 完整图表

桌面端 (> 1024px)
├── 多列布局
├── 固定侧边栏
├── 作文编辑器 + 批改结果 左右对照
└── 仪表盘多卡片网格
```
